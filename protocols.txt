
ALL PROTOCOLS are of the form client sends command responses and closes connection.
1 command, 1 response, connection closed.
* indicates to server where payload starts.

LOGIN protocol

client : LOGIN username password *
    server verifies username and password
    case 1 - correct and existing username and password
server : contents of note
    case 2 - invalid
server : INVALID
    server closes connection

LOGOUT protocol

client : LOGOFF username password *payload
    server verifies username and password
    case 1 - correct and existing username and password
    server stores password to file.
server : STORED
    case 2 - invalid
server : INVALID
    server closes connection

USERNAME protocol

client : USERNAME username password *payload
    server verifies username and password
    case 1 - correct and existing username and password
    updates username
server : STORED
    case 2 - invalid
server : INVALID
    server closes connection

PASSWORD protocol

client : PASSWORD username password *payload
    server verifies username and password
    case 1 - correct and existing username and password
    updates password
server : STORED
    case 2 - invalid
server : INVALID
    server closes connection

CREATE protocol
client : CREATE username password *
    case 1 - valid message
    server creates new profile
server : CREATED
    case 2 - already exisits
server : TAKEN
    server closes connection
